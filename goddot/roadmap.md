### 1.1 1v1 普攻对战（已完成）
**目标**：两个角色在极简战场上自动普攻到死。
**核心验证**：自动寻敌 → 移动 → 普攻 → 死亡 的最小闭环。
**任务**：
- 极简战场场景（两个出生点，足够距离）
- 角色数据：HP、攻击力、攻击速度、移动速度
- 自动行为：找最近的敌人 → 走到攻击范围 → 普攻 → 重复
- 死亡判定和战斗结束
**完结标准**：
- [x] 代码层闭环：BattleEngine + DamageCalculator + StrategyEvaluator 串联完毕
- [x] Main.cs 能加载数据并启动一场 1v1 战斗
- [x] 战斗结束有胜负判定（全灭 / AP耗尽按HP比例）
- [x] 控制台打印每回合战斗日志
- [x] CC框架接入：CharacterData/BattleUnit/Main.cs 支持CC状态切换，默认未CC
- [x] 12/12角色数据已填充（Subagent批量填充+修正），JSON格式验证通过
- [x] `CcInitialEquipmentIds` 配置：CC后新增装备槽强制带装备
- [x] `EquipmentSlot` 武器槽校验：MainHand永不为空
- [x] 运行 Godot 场景（未CC状态），验证实际输出符合预期
- [x] 一方 HP 归零，战斗结束

### 1.2 战前配置与 3v3 自动战斗（已完成）
**目标**：玩家在 Godot 控制台中完成队伍组建、阵型布置、被动技能配置、策略编程，启动 3v3 自动对战。
**核心验证**：战前配置全链路 → 自动战斗 → 被动技能触发 → 胜负判定。
**任务**：
- 控制台输入改为 Godot LineEdit UI（解决 Windows GUI stdin 问题）
- 队伍组建：从12角色池中选择3人组成己方队伍
- 阵型布置：为每个角色选择6位置之一（前排1-3，后排4-6）
- 敌人配置框架：enemy_formations.json + strategy_presets.json
- 被动技能选择系统：PP上限控制，玩家为每个角色装备被动
- 策略编程控制台：8条策略栏位（技能 + 条件1 + 条件2 + 模式）
- 条件系统MVP：SelfHp、SelfApPp、TargetSelector lowest/highest排序
- EventBus 接入 BattleEngine：6个战斗事件（BattleStart/BeforeActiveUse/BeforeHit/AfterHit/AfterActiveUse/BattleEnd）
- PassiveSkillProcessor：12个触发时机 + 同時発動制限 + 效果执行
- 3v3 完整对战验证
**完结标准**：
- [x] 控制台输入改为 Godot LineEdit UI，async/await 流程正常
- [x] 玩家可选3个己方角色并布置阵型
- [x] 敌方随机/预设阵型正确生成
- [x] 被动技能选择受 PP 上限约束
- [x] 策略编程支持8条栏位编辑、确认、恢复默认
- [x] 条件系统 MVP 可过滤和排序目标
- [x] EventBus 正确发布所有战斗事件
- [x] 被动技能在正确时机触发并打印实际效果
- [x] 3v3 自动战斗在 Godot 中可正常运行
- [x] 战斗日志显示剩余HP和debuff
- [x] 多轮对战支持（"是否再来一局"）

**【想法池】**

### >>> 1.3 被动技能完整实现与架构改造
**目标**：实现被动技能对伤害计算的干预、反击/追击/先制攻击、一次性标记系统。
**核心验证**：格挡/回避/必中/免疫/掩护类被动真正改变战斗结果；反击/追击在正确时机执行额外攻击。
**任务**：
- **模块1**：DamageCalculation 可变上下文（BeforeHitEvent 暴露 Calc 引用，DamageCalculator 读取可变参数）
- **模块2**：PendingActionQueue 行动队列（BattleEngine 核心循环支持插入反击/追击/先制攻击）
- **模块3**：TemporalState 临时标记系统（1次魔法无效、1hit变0、致死耐、必中下一次）
- **模块4**：友方目标选择 + 效果结构化（ApPlus1Ally、HealAlly、CoverAlly 等）
- **模块5**：逐个职业实现完整被动技能效果，填充 JSON 的 Effects 字段
**完结标准**：
- [ ] BeforeHit 阶段被动可以修改命中/回避/格挡/伤害/掩护目标
- [ ] 反击类被动（追击斩、广域反击）在友方被攻击后正确执行额外攻击
- [ ] 先制攻击类被动（快速打击）在战斗开始时正确执行
- [ ] 一次性免疫类被动（招架、魔法屏障）在对应攻击时消耗标记并免疫伤害
- [ ] 友方目标类被动（主动礼物、急救）正确选择友方目标并生效
- [ ] 所有30+被动技能的 Tags 迁移到结构化的 Effects 字段

**【想法池】**
- DamageCalculator 保留旧签名作为重载，确保过渡期内不破坏现有调用点
- 反击/追击不消耗 AP，威力/命中由被动技能定义
- 先制攻击在 BattleStartEvent 中入队，在战斗正式开始前执行
- 终结打击（战斗结束时攻击）需要在胜负判定前执行，因为可能改变胜负结果

## Phase 2：大巴扎循环
**大目标**：把战斗嵌入 Roguelike 爬塔结构，每次前进=抽牌=成长。

### 2.1 最小地图循环
**目标**：战斗 → 奖励 → 再战斗。
**任务**：
- 极简节点地图（3-5个节点串联）
- 节点类型：战斗、休息（回血）、奖励
- 战斗奖励：获得一个新角色/技能/道具
- 角色/技能/道具的获取和装备逻辑
**完结标准**：
- [ ] 能走完整的一小轮地图
- [ ] 战斗胜利后能获得并装备新东西
- [ ] 新东西能影响下一场战斗

### 2.2 抽牌与天数系统
**目标**：完整的大巴扎式抽牌驱动流程。
**任务**：
- "抽牌"机制：每次前进从牌池抽一个节点
- 天数限制：总天数/行动点，耗尽则本轮结束
- 牌池设计：不同稀有度、不同效果
- 经济系统：金币/资源用于购买、升级
**完结标准**：
- [ ] 每次前进有抽牌感
- [ ] 天数压力形成决策张力
- [ ] 一轮游戏的开始→中期→后期有体验变化

### 2.3 局外成长（Meta）
**目标**：反复游玩有价值。
**任务**：
- 解锁新角色、新技能加入未来牌池
- 难度选择
- 成就/统计
**完结标准**：
- [ ] 玩完一轮后，下一轮有新内容
