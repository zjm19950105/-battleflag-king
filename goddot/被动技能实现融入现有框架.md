# 被动技能实现融入现有框架 — 架构修改方案

> **重要性**：此文档记录被动技能系统对现有框架的侵入式改造方案。被动技能不是"锦上添花"的装饰，而是战斗核心循环的有机组成部分（格挡/回避/反击/追击/先制攻击）。如果不按此方案改造框架，大量被动技能将无法实现或只能打印日志假触发。
>
> **状态**：设计完成，待实施  
> **最后更新**：2026-05-05  
> **依据**：《unicorn-overlord-passive-skills-complete.md》+ 当前代码基线

---

## 一、当前框架现状（基线）

### 1.1 战斗事件流（BattleEngine.ExecuteUnitTurn）

```
BeforeActiveUseEvent
  ↓ 消耗AP
foreach target:
  BeforeHitEvent
    ↓
  DamageCalculator.Calculate(attacker, defender, skill, ctx)
    → 内部创建 DamageCalculation（不可外部修改）
    → 固定流程：命中判定 → 回避判定 → 暴击判定 → 格挡判定
    → 返回 DamageResult（只读）
    ↓
  AfterHitEvent
    ↓
AfterActiveUseEvent
```

**问题**：`BeforeHitEvent` 没有任何可被修改的计算参数。`DamageCalculator` 是一个黑盒，被动技能无法干预命中/回避/格挡/伤害数值。

### 1.2 被动技能处理器（PassiveSkillProcessor）

- 通过 `EventBus` 订阅 6 个事件
- `ProcessTiming` / `ProcessForUnit` 遍历单位，匹配 `TriggerTiming`
- 用 `string` tags 做 switch-case 判断效果类型
- 当前只实现了：自身 BUFF、自身 AP/HP 回复

**问题**：
1. 无法干预伤害计算（格挡/回避/必中/伤害免疫）
2. 无法对友方释放（AP+1 友方、HP 回复友方）
3. 无法触发额外攻击（反击/追击/先制）
4. 没有一次性状态管理（1 次魔法无效、1hit 伤害变 0）
5. Tags 系统和实际效果描述脱节（JSON 中 tags 大量为占位符）

### 1.3 数据层

- `PassiveSkillData` 有 `Tags: List<string>` 和 `Effects: List<SkillEffectData>`
- 当前 `Effects` 字段在 JSON 中为空数组
- `SkillEffectData` 已有 `{ EffectType, Parameters }` 结构，但未使用

### 模块 1：DamageCalculation 可变上下文（最高优先级）

**目标**：让被动技能能在 `BeforeHit` 阶段读取和修改伤害计算的所有参数。

#### 3.1.2 DamageCalculation 扩展

在现有属性基础上新增：

```csharp
public class DamageCalculation
{
    // 已有属性...
    public int FinalAttackPower { get; set; }
    public int FinalDefense { get; set; }
    public float SkillPowerRatio { get; set; } = 1.0f;
    public bool IsHit { get; set; } = true;
    public bool IsCritical { get; set; } = false;
    public bool IsBlocked { get; set; } = false;
    public bool IsEvaded { get; set; } = false;
    public float CritMultiplier { get; set; } = 1.5f;
    public float BlockReduction { get; set; } = 0f;
    public int PhysicalDamage { get; set; }
    public int MagicalDamage { get; set; }
    public List<StatusAilment> AppliedAilments { get; set; } = new();

    // === 新增：被动技能可修改的参数 ===

    /// <summary>强制命中（无视命中判定，但无法无视回避技能）</summary>
    public bool ForceHit { get; set; } = false;

    /// <summary>强制回避（必定回避，无视黑暗和命中）</summary>
    public bool ForceEvasion { get; set; } = false;

    /// <summary>强制格挡（必定格挡）或强制不可格挡</summary>
    public bool? ForceBlock { get; set; } = null;

    /// <summary>技能威力倍率（死亡之刃、不屈精神等）</summary>
    public float SkillPowerMultiplier { get; set; } = 1.0f;

    /// <summary>最终伤害倍率</summary>
    public float DamageMultiplier { get; set; } = 1.0f;

    /// <summary>无视防御比例（0~1，如"重装系无视 100%防御"）</summary>
    public float IgnoreDefenseRatio { get; set; } = 0f;

    /// <summary>物理伤害免疫（如"1hit 变 0"、"1 次物理无效"）</summary>
    public bool NullifyPhysicalDamage { get; set; } = false;

    /// <summary>魔法伤害免疫</summary>
    public bool NullifyMagicalDamage { get; set; } = false;

    /// <summary>掩护目标：如果不为 null，攻击将转移至此目标</summary>
    public BattleUnit CoverTarget { get; set; } = null;

    /// <summary>本次攻击不可被掩护</summary>
    public bool CannotBeCovered { get; set; } = false;

    /// <summary>攻击不可被格挡（ガード不可）</summary>
    public bool CannotBeBlocked { get; set; } = false;
}
```

#### 3.1.3 DamageCalculator 改造

签名从：
```csharp
public DamageResult Calculate(BattleUnit attacker, BattleUnit defender, ActiveSkill skill, BattleContext ctx)
```

改为：
```csharp
public DamageResult Calculate(DamageCalculation calc)
```

**流程改造**：

```csharp
public DamageResult Calculate(DamageCalculation calc)
{
    var attacker = calc.Attacker;
    var defender = calc.Defender;
    var skill = calc.Skill;

    // Stage 1-2: 攻防计算（已有）
    calc.FinalAttackPower = attacker.GetCurrentAttackPower(skill.Type);
    calc.FinalDefense = defender.GetCurrentDefense(skill.Type) * (1 - calc.IgnoreDefenseRatio);

    // Stage 3-6: 基础伤害 × 技能威力 × 特性补正
    float baseDmg = calc.BaseDifference * (skill.Power / 100f * calc.SkillPowerMultiplier);
    baseDmg *= GetClassTraitMultiplier(attacker, defender);
    baseDmg *= GetCharacterTraitMultiplier(attacker, defender, skill);

    // 分物理/魔法伤害（已有）...

    // Stage 7: 命中判定（改造后）
    if (!calc.ForceHit)
    {
        if (!RollHit(attacker, defender, skill)) // 内部读取 calc 的状态
        {
            calc.IsHit = false;
            ZeroDamage(calc);
            return BuildResult(calc);
        }
    }

    // Stage 8: 回避判定（改造后）
    if (calc.ForceEvasion || RollEvasion(attacker, defender, skill))
    {
        calc.IsEvaded = true;
        ZeroDamage(calc);
        return BuildResult(calc);
    }

    // === 掩护处理 ===
    if (calc.CoverTarget != null && !calc.CannotBeCovered)
    {
        defender = calc.CoverTarget;
        calc.Defender = defender;
        // 重新计算防御（因为掩护者防御不同）
        calc.FinalDefense = defender.GetCurrentDefense(skill.Type);
    }

    // Stage 9: 暴击（已有）...

    // Stage 10: 格挡（改造后）
    if (skill.HasPhysicalComponent && calc.ForceBlock != false && !calc.CannotBeBlocked)
    {
        if (calc.ForceBlock == true || RollBlock(defender, skill))
        {
            calc.IsBlocked = true;
            calc.BlockReduction = defender.GetBlockReduction();
            calc.PhysicalDamage = (int)(calc.PhysicalDamage * (1 - calc.BlockReduction));
        }
    }

    // === 伤害免疫处理 ===
    if (calc.NullifyPhysicalDamage) calc.PhysicalDamage = 0;
    if (calc.NullifyMagicalDamage) calc.MagicalDamage = 0;

    // === 最终伤害倍率 ===
    calc.PhysicalDamage = (int)(calc.PhysicalDamage * calc.DamageMultiplier);
    calc.MagicalDamage = (int)(calc.MagicalDamage * calc.DamageMultiplier);

    // 四舍五入...
    return BuildResult(calc);
}
```

### 模块 2：行动队列系统（PendingActionQueue）

**目标**：支持反击、追击、先制攻击、战斗结束时攻击等"额外行动"。

#### 3.2.1 核心设计

```csharp
public enum PendingActionType
{
    Preemptive,   // 战斗开始时先制攻击
    Counter,      // 被攻击后反击
    Pursuit,      // 友方攻击时追击
    PostAction,   // 自身攻击后追击（快速装填）
    BattleEnd,    // 战斗结束时攻击（终结打击）
    ExtraTurn     // 行动顺序最速（快速施法）
}

public class PendingAction
{
    public PendingActionType Type { get; set; }
    public BattleUnit Actor { get; set; }
    public List<BattleUnit> Targets { get; set; }

    // 攻击参数（被动触发的攻击通常不耗 AP）
    public int Power { get; set; }
    public int? HitRate { get; set; }
    public SkillType DamageType { get; set; }
    public AttackType AttackType { get; set; }
    public TargetType TargetType { get; set; }

    // 特殊标签
    public List<string> Tags { get; set; } = new(); // SureHit, CannotBeBlocked, Ranged...

    // 触发来源（用于日志）
    public string SourcePassiveId { get; set; }
}
```

#### 3.2.2 BattleEngine 核心循环改造

当前循环：
```csharp
while (true)
{
    var turnOrder = ...;
    foreach (var unit in turnOrder)
    {
        ExecuteUnitTurn(unit);
    }
}
```

改造后：
```csharp
public class BattleEngine
{
    private Queue<PendingAction> _pendingActions = new();

    public void EnqueueAction(PendingAction action) => _pendingActions.Enqueue(action);

    public BattleResult StartBattle()
    {
        // 战斗开始时：处理先制攻击队列
        _eventBus.Publish(new BattleStartEvent { Context = _ctx });
        ProcessPendingActions(); // 快速打击、骑士打击等先制攻击

        while (true)
        {
            // ... 回合循环 ...
            foreach (var unit in turnOrder)
            {
                if (!unit.IsAlive) continue;
                ExecuteUnitTurn(unit);
                ProcessPendingActions(); // 处理本回合产生的反击/追击
            }

            ProcessPendingActions(); // 回合结束后处理

            // ... 胜负判定 ...
        }
    }

    private void ProcessPendingActions()
    {
        while (_pendingActions.Count > 0)
        {
            var action = _pendingActions.Dequeue();
            if (!action.Actor.IsAlive) continue;

            // 构建临时 ActiveSkill 上下文
            var tempSkill = CreateTempSkill(action);
            foreach (var target in action.Targets.Where(t => t.IsAlive))
            {
                ExecutePendingAttack(action.Actor, target, tempSkill, action);
            }
        }
    }

    private void ExecutePendingAttack(BattleUnit attacker, BattleUnit defender, ActiveSkill skill, PendingAction action)
    {
        var calc = new DamageCalculation { Attacker = attacker, Defender = defender, Skill = skill };
        _eventBus.Publish(new BeforeHitEvent { ... Calc = calc });
        var result = _damageCalculator.Calculate(calc);

        if (result.IsHit && !result.IsEvaded)
            defender.TakeDamage(result.TotalDamage);

        _eventBus.Publish(new AfterHitEvent { ... });
    }
}
```

**关键决策**：
- 反击/追击不消耗 AP
- 反击/追击的目标通常由被动技能描述决定（如"敌单体反击"→攻击原攻击者）
- 先制攻击在 `BattleStartEvent` 处理时加入队列，在战斗正式开始前执行
- 终结打击在 `BattleEndEvent` 处理时加入队列，但需要在胜负判定"之前"执行（因为终结打击可能改变胜负结果）。实际上 wiki 说"战斗结束时"是在战斗结束前发动，所以需要在 `EndBattle` 之前处理。

### 模块 3：临时状态标记系统（TemporalState）

**目标**：管理"1 次免疫"、"N hit 无效"、"致死耐"等有时限或次数限制的状态。

#### 3.3.1 设计

```csharp
public class TemporalState
{
    public string Key { get; set; }           // 如 "MagicBarrier", "MeleeNullify", "DeathResist"
    public int RemainingCount { get; set; }   // 剩余次数（-1 表示无限）
    public int RemainingTurns { get; set; }   // 剩余回合（-1 表示无限，0 表示本回合）
    public string SourceSkillId { get; set; }
}
```

#### 3.3.2 BattleUnit 扩展

```csharp
public List<TemporalState> TemporalStates { get; private set; } = new();

public bool TryConsumeTemporal(string key)
{
    var state = TemporalStates.FirstOrDefault(s => s.Key == key && s.RemainingCount != 0);
    if (state == null) return false;
    if (state.RemainingCount > 0) state.RemainingCount--;
    if (state.RemainingCount == 0) TemporalStates.Remove(state);
    return true;
}

public void AddTemporal(string key, int count = -1, int turns = -1, string source = null)
{
    TemporalStates.Add(new TemporalState { Key = key, RemainingCount = count, RemainingTurns = turns, SourceSkillId = source });
}
```

#### 3.3.3 使用场景

| 被动技能 | 时机 | Temporal Key | 消耗点 |
|---------|------|-------------|--------|
| 魔法屏障 | 友方被魔法攻击前 | `MagicDamageNullify` | BeforeHit 检查并消耗 |
| 招架 | 被近接攻击直前 | `MeleeHitNullify` | BeforeHit 检查并消耗 |
| 致死耐 buff | 战斗开始时/攻击前 | `DeathResist` | 受到致死伤害时检查 |
| 纯净领域 | 战斗开始时 | `DebuffNullify` | 被附加 debuff 时检查并消耗 |
| 王者塔盾 | 战斗开始时 | `PhysicalDamageNullify` | BeforeHit 检查并消耗 |
| 神龙缰绳 | 主动攻击前 | `ForceHitNext` | BeforeHit 检查并消耗 |

### 模块 4：友方目标选择 + 效果结构化

**目标**：让被动技能可以正确选择友方目标，并统一效果执行逻辑。

#### 3.4.1 效果结构化（从 Tag 迁移到 Effects）

当前 JSON 中 `Effects` 为空，需要填充。建议格式：

```json
{
  "id": "pas_give_ap",
  "name": "主动礼物",
  "effects": [
    { "effectType": "RecoverAp", "parameters": { "target": "randomAlly", "amount": 1 } }
  ]
}
```

常用 EffectType 清单：

```
RecoverAp / RecoverPp / RecoverHp      // 回复类
AddBuff / AddDebuff                     // BUFF/DEBUFF 类（含目标选择）
ApplyAilment                            // 异常状态
RecoverAilment                          // 解除异常
TemporalMark                            // 添加一次性标记
CounterAttack                           // 反击（生成 PendingAction）
PursuitAttack                           // 追击
ModifyDamageCalc                        // 修改伤害计算参数（BeforeHit 用）
ForceHit / ForceEvasion                 // 强制命中/回避
CoverAlly                               // 掩护（修改 Calc.CoverTarget）
ModifySkillPower                        // 调整技能威力
DrainHp                                 // 吸血
ReflectDamage                           // 反伤/反射
ShiftTarget                             // 改变攻击目标
```

#### 3.4.2 目标选择扩展

被动技能的目标类型：

```csharp
public enum PassiveTargetType
{
    Self,           // 自己
    Attacker,       // 攻击者（反击时用）
    Defender,       // 被攻击者（掩护时用）
    RandomAlly,     // 随机友方
    LowestHpAlly,   // HP 最低友方
    HighestHpAlly,
    AllAllies,      // 全体友方
    ColumnAllies,   // 同列友方
    RowAllies,      // 同排友方
    FrontRowAlly,   // 前排友方
    BackRowAlly,
    SameClassAlly   // 同兵种友方（如"同列骑乘系"）
}
```

`PassiveSkillProcessor` 需要引入一个简化的目标选择器：

```csharp
private List<BattleUnit> SelectPassiveTargets(
    BattleUnit owner, PassiveTargetType targetType, BattleContext ctx, BattleUnit contextUnit = null)
{
    var allies = GetAllies(owner, ctx);
    return targetType switch
    {
        PassiveTargetType.Self => new List<BattleUnit> { owner },
        PassiveTargetType.Attacker => new List<BattleUnit> { contextUnit },
        PassiveTargetType.Defender => new List<BattleUnit> { contextUnit },
        PassiveTargetType.LowestHpAlly => new List<BattleUnit> { allies.OrderBy(u => u.CurrentHp).First() },
        PassiveTargetType.AllAllies => allies,
        PassiveTargetType.ColumnAllies => allies.Where(u => u.Position == contextUnit?.Position).ToList(),
        // ...
    };
}
```

### 模块 5：事件系统扩展

**目标**：覆盖 wiki 中所有发动时机。

#### 3.5.1 新增/明确事件定义

| 事件名 | 触发点 | 说明 |
|--------|--------|------|
| `BattleStartEvent` | 已存在 | 战斗开始时 |
| `BattleEndEvent` | 已存在 | 战斗结束时（但要在终结打击等执行后再判定胜负） |
| `BeforeActiveUseEvent` | 已存在 | 自身使用主动前 |
| `AfterActiveUseEvent` | 已存在 | 自身使用主动后（所有目标攻击完成后） |
| `BeforeHitEvent` | 已存在（需扩展 Calc） | 每次攻击单个目标前 |
| `AfterHitEvent` | 已存在 | 每次攻击单个目标后 |
| `OnBeingHitEvent` | **建议新增** | 从"被攻击者"视角的 AfterHit。当前 AfterHit 可以共用，但最好有区分视角的明确命名 |
| `BeforeAllyAttackEvent` | **建议新增** | 其他友方主动攻击前（锐利呐喊、魔法剑刃） |
| `OnAllyActiveUseEvent` | **建议新增** | 其他友方使用主动时（主动礼物、快速行动） |
| `OnAllyAttackedEvent` | **建议新增** | 友方被攻击时（追击斩、狂暴） |
| `BeforeAllyHitEvent` | **建议新增** | 友方被攻击前（光辉掩护、重掩护） |
| `BeforePhysicalHitEvent` | 已存在（ProcessTiming 中） | 被物理攻击直前 |
| `BeforeMeleeHitEvent` | 已存在（ProcessTiming 中） | 被近接攻击直前 |
| `AfterActionEvent` | 已存在 | 行动后（敌我全员） |

**关键决策**：
- `BeforeAllyAttackEvent` 和 `BeforeActiveUseEvent` 的区别：前者是"别人"攻击时触发，后者是"自己"攻击时触发。但 wiki 中"自身使用主动时"和"自身主动攻击前"在实际代码中可以用同一个事件区分（通过条件判断触发者）。
- 实际上更简洁的做法是：**不新增过多事件类型**，而是在 `PassiveSkillProcessor` 中统一订阅 `BeforeActiveUseEvent`、`AfterHitEvent` 等，然后通过 `TriggerTiming` + `条件判断` 来区分。当前架构已经是这样做的。
- 真正需要新增的事件：`OnAllyAttackedEvent`（友方被攻击时）和 `BeforeAllyHitEvent`（友方被攻击前），因为当前 `AfterHitEvent`/`BeforeHitEvent` 只从攻击者角度发布一次，被动处理器需要扫描友方来找触发者。

不过实际上，当前的 `AfterHitEvent` 已经包含了 `Attacker` 和 `Defender`，被动处理器在 `OnAfterHit` 中遍历友方来查找是否有带有 `AllyOnAttacked` 时机的技能，这是可行的。

所以事件系统**不需要大量新增**，保持当前 6 个核心事件即可，被动处理器在事件回调中做友方扫描。

## 六、关键设计决策记录（供未来参考）

### 决策 1：为什么 DamageCalculation 必须是可变引用？

 damage calculation 流水线有 10 个阶段，被动技能可能干预其中任何阶段：
- 命中前：必中（跳过 Stage 7）
- 回避前：回避技能（跳过 Stage 8）
- 格挡前：提升格挡率/强制格挡（修改 Stage 10）
- 伤害计算前：提升威力/无视防御（修改 Stage 3-6）
- 伤害计算后：伤害免疫（修改 Stage 10 之后）

如果每个干预点都单独发一个事件（如 `BeforeHitCheckEvent`、`BeforeEvasionEvent`...），事件系统会爆炸。让 `DamageCalculation` 成为贯穿整个计算流程的可变上下文，是唯一简洁的方案。

### 决策 2：反击为什么不走正常的 ExecuteUnitTurn？

反击/追击/先制攻击的特点是：
1. 不消耗 AP
2. 通常只有一个目标
3. 威力/命中由被动技能定义，不是角色当前装备的技能
4. 发生在正常回合之外

如果复用 `ExecuteUnitTurn`，需要大量 `if (isCounterAttack)` 的特殊判断。不如单独走 `ExecutePendingAttack` 流程，复用 `BeforeHit → Calculate → AfterHit` 的核心攻击管线即可。

### 决策 3：TemporalState 为什么不用 Buff 系统？

Buff 系统的设计是"持续若干回合的数值修改"，而 TemporalState 是"特定条件触发 N 次后消失"。
- 魔法屏障：不是数值修改，是"下一次魔法攻击时免疫"
- 致死耐：不是防御力提升，是"受到致死伤害时保留 1HP"
- 招架：不是常驻格挡率提升，是"下一次近接攻击时完全无效"

这些效果无法用 Buff 的 `RemainingTurns` + `Ratio` 模型表达。必须引入单独的计数器/标记系统。

### 决策 4：为什么保持 Tags 同时引入 Effects？

Tags 用于标识"特性"（如 `Ranged`、`CannotBeBlocked`、`SimultaneousLimit`），这些不是具体执行的效果，而是攻击/技能的元属性。Effects 用于标识"具体做什么"。两者并存，Tags 决定"怎么打"，Effects 决定"打出什么结果"。

## 七、当前 MVP 的临时兼容策略

在完整改造完成前，当前代码可以保持运行：

1. `DamageCalculator` 保留旧签名作为重载：

   public DamageResult Calculate(BattleUnit a, BattleUnit d, ActiveSkill s, BattleContext c)
