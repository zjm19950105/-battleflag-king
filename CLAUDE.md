## 架构概览

```
src/
  core/        # 核心玩法机制（格子、单位、战斗、AI）
  systems/     # 支持系统（输入、存档、音频、网络）
  ui/          # 用户界面（HUD、菜单、对话框）
  data/        # 数据配置（单位属性、关卡、技能表）
  utils/       # 通用工具（数学、事件总线、对象池）
```

### 模块职责
| 模块 | 职责 | 关键概念 |
|------|------|----------|
| `core` | 战棋核心规则 | Grid, Unit, BattleEngine, TurnManager |
| `systems` | 跨模块服务 | InputManager, SaveSystem, AudioManager |
| `ui` | 画面呈现 | 不直接修改游戏状态，只响应事件 |
| `data` | 静态配置 | JSON / 配置表驱动，代码中无硬编码数值 |
| `utils` | 辅助设施 | EventBus, Pathfinder, RandUtil |

## 接口契约
- **模块间通信**：优先通过 `EventBus`（发布-订阅），避免直接引用
- **数据流向**：`Input → Systems → Core → UI`，Core 是唯一直接修改游戏状态的地方
- **配置加载**：`data/` 下的配置在启动时一次性加载为只读对象，运行时不修改文件
