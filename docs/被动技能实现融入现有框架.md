# 被动技能旧方案归档

最后更新：2026-05-08

这个文件曾是 Phase 1.3 的被动技能改造方案。当前实现已经进入 A01-A16 之后的状态，实际代码以这些文件为准：

- `goddot/src/Skills/PassiveSkillProcessor.cs`
- `goddot/src/Skills/SkillEffectExecutor.cs`
- `goddot/src/core/BattleEngine.cs`
- `docs/csharp-architecture.md`

当前原则：

- `PassiveSkillProcessor` 只负责触发时机、条件、PP、同时发动限制和需要战斗阶段上下文的仲裁。
- 通用效果原子优先委派给 `SkillEffectExecutor`。
- 不要重新引入旧的空 `ISkillEffect.Apply()` 双轨。
- legacy tag-only 被动只能作为迁移遗留存在；新增战斗语义必须写 structured `effects`。

如果要继续拆被动系统，请先补测试，再按 `roadmap.md` 的 Phase 1.4-B 做小步重构。
