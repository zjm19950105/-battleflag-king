### 1.2 战前配置与 3v3 自动战斗（已完成）
**目标**：玩家在 Godot 控制台中完成队伍组建、阵型布置、被动技能配置、策略编程，启动 3v3 自动对战。
**核心验证**：战前配置全链路 → 自动战斗 → 被动技能触发 → 胜负判定。
**任务**：
- 控制台输入改为 Godot LineEdit UI（解决 Windows GUI stdin 问题）
- 队伍组建：从12角色池中选择3人组成己方队伍
- 阵型布置：为每个角色选择6位置之一（前排1-3，后排4-6）
- 敌人配置框架：enemy_formations.json + strategy_presets.json
- 被动技能选择系统：PP上限控制，玩家为每个角色装备被动
- 策略编程控制台：8条策略栏位（技能 + 条件1 + 条件2 + 模式）
- 条件系统MVP：SelfHp、SelfApPp、TargetSelector lowest/highest排序
- EventBus 接入 BattleEngine：6个战斗事件（BattleStart/BeforeActiveUse/BeforeHit/AfterHit/AfterActiveUse/BattleEnd）
- PassiveSkillProcessor：12个触发时机 + 同時発動制限 + 效果执行
- 3v3 完整对战验证
**完结标准**：
- [x] 控制台输入改为 Godot LineEdit UI，async/await 流程正常
- [x] 玩家可选3个己方角色并布置阵型
- [x] 敌方随机/预设阵型正确生成
- [x] 被动技能选择受 PP 上限约束
- [x] 策略编程支持8条栏位编辑、确认、恢复默认
- [x] 条件系统 MVP 可过滤和排序目标
- [x] EventBus 正确发布所有战斗事件
- [x] 被动技能在正确时机触发并打印实际效果
- [x] 3v3 自动战斗在 Godot 中可正常运行

### 1.3 被动技能完整实现与架构改造（已完成）
**目标**：实现被动技能对伤害计算的干预、反击/追击/先制攻击、一次性标记系统；同时补齐主动技能所需的框架支持（多段攻击、击倒事件、蓄力）。
**核心验证**：格挡/回避/必中/免疫/掩护类被动真正改变战斗结果；反击/追击在正确时机执行额外攻击；多段攻击每hit独立判定。
**任务**：
- **模块1**：DamageCalculation 可变上下文 + **多段攻击支持**（BeforeHitEvent 暴露 Calc 引用，DamageCalculator 支持 N-hit 循环，每hit独立判定命中/格挡/暴击）
- **模块2**：PendingActionQueue 行动队列（BattleEngine 核心循环支持插入反击/追击/先制攻击）
- **模块3**：TemporalState 临时标记系统（1次魔法无效、1hit变0、致死耐、必中下一次）
- **模块4**：友方目标选择 + 效果结构化 + **自定义计数器**（通用 `Dictionary<string,int>` 支持小精灵等自定义资源）
- **模块5**：逐个职业实现完整被动技能效果，填充 JSON 的 Effects 字段
- **模块6** 🆕：OnKnockdown 事件 + Charge 蓄力状态机 + 动态技能威力（HP条件/位置条件/buff条件）
**完结标准**：
- [x] BeforeHit 阶段被动可以修改命中/回避/格挡/伤害/掩护目标
- [x] 多段攻击（2~9hit）每hit独立跑伤害流水线，第1hit被格挡可影响后续效果
- [x] 反击类被动（追击斩、广域反击）在友方被攻击后正确执行额外攻击
- [x] 先制攻击类被动（快速打击）在战斗开始时正确执行
- [x] 一次性免疫类被动（招架、魔法屏障）在对应攻击时消耗标记并免疫伤害
- [x] 友方目标类被动（主动礼物、急救）正确选择友方目标并生效
- [x] 击倒时触发效果（AP+1、PP+1、HP回复等）正确执行
- [x] 蓄力技能第1回合蓄力（回避/被动不可用），第2回合发动
- [x] 所有30+被动技能的 Tags 迁移到结构化的 Effects 字段

- [x] 7新角色（→18角色）：巫师/女巫/白骑士/狮鹫骑士/萨满/角斗士/精灵女先知
- [x] ConditionEvaluator 3空壳补全：AttributeRank/TeamSize/AttackAttribute
- [x] 全部技能数据修正：24被动tags + 3效果错误 + 4互换tags + 3多余删除
- [x] 装备系统：80件装备（8大类型），战前装备配置UI + EquipmentSetup阶段
- [x] 策略条件编辑器：12类条件级联选择 + 仅/优先模式，8条策略栏位完整编辑
- [x] 被动技能条件策略：每个被动可设发动条件，PassiveSkillProcessor触发前检查
- [x] 敌方策略手动配置：我方→敌方依次
- [x] AP/PP星星显示 + 上一步按钮 + 战斗状态面板增强(buff/debuff/异常/标记/CC/兵种颜色)


**【想法池】**
- DamageCalculator 保留旧签名作为重载，确保过渡期内不破坏现有调用点
- 反击/追击不消耗 AP，威力/命中由被动技能定义
- 先制攻击在 BattleStartEvent 中入队，在战斗正式开始前执行
- 终结打击（战斗结束时攻击）需要在胜负判定前执行，因为可能改变胜负结果
- 🔒 **资源偷取（AP/PP转移）**：被动偷取、主动偷取、战姬长枪等技能暂时锁定。原因：需要先确定大巴扎奖励系统中的资源获取机制，才能设计跨战斗的资源转移。技能数据保留在 JSON 中但标记为 `"locked": true`
- ✅ **自定义计数器系统**：已并入模块4设计。`BattleUnit.CustomCounters: Dictionary<string, int>` 通用键值对，支持小精灵/怒气/连击等任意计数器。`DamageCalculation.CounterPowerBonus` 处理计数器对伤害的加成。EffectType 新增 `ModifyCounter` / `ConsumeCounter`
- 🚫 **勇气技能全部排除**：参考文档中的"勇气"类型技能不属于本项目范围，不纳入数据填充

## Phase 2：大巴扎循环
**大目标**：把战斗嵌入 Roguelike 爬塔结构，每次前进=抽牌=成长。

### 2.1 最小地图循环
**目标**：战斗 → 奖励 → 再战斗。
**任务**：
- 极简节点地图（3-5个节点串联）
- 节点类型：战斗、休息（回血）、奖励
- 战斗奖励：获得一个新角色/技能/道具
- 角色/技能/道具的获取和装备逻辑
**完结标准**：
- [ ] 能走完整的一小轮地图
- [ ] 战斗胜利后能获得并装备新东西
- [ ] 新东西能影响下一场战斗

### 2.2 抽牌与天数系统
**目标**：完整的大巴扎式抽牌驱动流程。
**任务**：
- "抽牌"机制：每次前进从牌池抽一个节点
- 天数限制：总天数/行动点，耗尽则本轮结束
- 牌池设计：不同稀有度、不同效果
- 经济系统：金币/资源用于购买、升级
**完结标准**：
- [ ] 每次前进有抽牌感
- [ ] 天数压力形成决策张力
- [ ] 一轮游戏的开始→中期→后期有体验变化

### 2.3 局外成长（Meta）
**目标**：反复游玩有价值。
**任务**：
- 解锁新角色、新技能加入未来牌池
- 难度选择
- 成就/统计
**完结标准**：
- [ ] 玩完一轮后，下一轮有新内容
